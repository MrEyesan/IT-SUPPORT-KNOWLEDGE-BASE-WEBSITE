1. Purpose & Scope

This manual documents standardized, secure procedures for resetting user passwords across corporate-managed systems. It is intended for IT Service Desk staff and system administrators. The goal is to minimize downtime while protecting user identity and complying with security policies.

2. Roles & Responsibilities

  • Service Desk Agent (Tier 1): Primary point for password reset requests, performs verification steps, executes standard resets, and logs actions.
  • Tier 2 Support / AD Admin: Handles complex resets, bulk changes, privileged account resets, and PowerShell/automation help.
  • Security / IAM Team: Approves escalation for privileged accounts, audits logs, and manages SSPR policies.
  • Manager / Sponsor: Authorizes emergency or out-of-policy password resets when required.

3. Security & Verification Policy

Before performing any password reset, agents MUST verify the user identity using at least two independent factors from the approved list. Examples (choose any two):

  • Company email address confirmation (from a known device) — only after verifying via an alternate channel.
  • Verified alternate email on file.
  • Verified phone number on file (call-back or OTP to recorded number).
  • Knowledge-based questions previously established in HR onboarding.
  • In-person ID check (if on campus).
  • Manager approval (email from manager’s corporate account).

Strictly prohibited: Accepting social media DMs, SMS from unknown numbers, remote control without explicit documented consent, or any single-factor identity confirmation. Agents must record the verification steps in the ticket (who, which factors, timestamp).

4. Password Policy (summary)

  • Minimum length: 12 characters.
  • Complexity: Mix of upper and lower case letters, numbers, and symbols. (Company may allow passphrases.)
  • Prohibited: Common passwords, dictionary words, sequential patterns.
  • Expiration: 180 days (if applicable).
  • MFA: All accounts must have MFA enabled where supported. Password reset should not bypass MFA.

5. Supported Systems & Quick Reference

  • On-prem Active Directory (AD) — primary corporate directory for domain-joined Windows accounts.
  • Azure AD / Microsoft 365 — cloud identity for Office 365, Intune-enrolled devices.
  • Local Windows account — device-specific local accounts
  • macOS local accounts — company-managed or BYOD, where supported
  • Linux accounts — local / LDAP-managed
  • Third-party SaaS — Salesforce, Slack, GitHub, etc. (handled per vendor guidance)

6. Standard Password Reset Workflows

6.1 Self-Service Password Reset (SSPR)

Encourage users to register for SSPR (company policy will define mandatory registration). SSPR flow: user visits the company SSPR portal -> verifies via registered factor(s) -> sets new password -> forced password complexity check -> optionally requires MFA re-enrollment. Document where to find SSPR logs and how to validate (portal, Azure AD sign-in logs).

6.2 Help Desk Assisted Reset (Verified)

The agent verifies user identity using two approved factors. Agent resets password in authoritative system (AD/Azure AD) following documented steps. Agent forces password change at next logon (where applicable) and optionally sets temporary password rules. Document action in the ticket and provide steps for the user to re-enroll in MFA or update credentials on devices.

6.3 Emergency / Privileged Account Resets

Require Tier 2 or IAM approval and a manager sign-off unless covered by a predefined emergency runbook. Reset must include an immediate forced re-authentication and a post-reset security review.

6.4 Account Unlocks

If the user is locked out due to too many failed attempts, verify identity and unlock the account without changing the password if permitted.

7. Step-by-step Procedures

NOTE: Always follow the Verification Policy before performing any of the steps below.

7.1 Active Directory (AD) — GUI (ADUC)

  1. Open Active Directory Users and Computers (ADUC).
  2. Find the user (use preconfigured saved queries when available).
  3. Right-click the user -> Reset Password.
  4. Enter a temporary password that meets policy, or check User must change password at next logon.
  5. Optionally, check the Unlock account if locked.
  6. Click OK.

Document the action in the ticket: who reset, temporary password policy, verification factors used.

7.2 Active Directory — PowerShell (examples)

Prerequisite: RSAT / Active Directory module installed, and you have delegated rights.

Reset password (example):

# Example: reset password and force change at next logon
$plaintext = 'TempP@ssw0rd123!'
$secure = ConvertTo-SecureString $plaintext -AsPlainText -Force
Set-ADAccountPassword -Identity 'j.smith' -NewPassword $secure -Reset
Set-ADUser -Identity 'j.smith' -ChangePasswordAtLogon $true
Unlock-ADAccount -Identity 'j.smith' # if locked

Notes: Replace 'j.smith' with the user's sAMAccountName or distinguished name. Confirm you have Reset Password delegated rights for the OU.

7.3 Azure AD / Microsoft 365

Option A — MSOnline module (example):

Connect-MsolService
Set-MsolUserPassword -UserPrincipalName user@company.com -NewPassword 'TempP@ssw0rd123!' -ForceChangePassword $true

Option B — AzureAD module (example):

Connect-AzureAD
# Example: set password profile via AzureAD (replace ObjectId with user ObjectId)
$passwordProfile = New-Object -TypeName Microsoft.Open.AzureAD.Model.PasswordProfile
$passwordProfile.Password = 'TempP@ssw0rd123!'
$passwordProfile.ForceChangePasswordNextLogin = $true
Set-AzureADUserPassword -ObjectId <userObjectId> -PasswordProfile $passwordProfile

Important: Module names and cmdlets vary by module and Microsoft updates — confirm the correct module for your environment.

7.4 macOS local account

Confirm user identity. If FileVault is enabled and password reset via IDMS is configured, follow the IDMS flow. For local reset (on device): boot to Recovery -> open Terminal -> use resetpassword utility, or use MDM solution to push password reset action. If using MDM (Jamf / Intune), use the console to clear the password and force the user to set a new one.

7.5 Linux (local account)

Local account (root or sudo required):

# As root or using sudo
sudo passwd username
# Follow prompts to set the new password

If PAM/LDAP is used, follow LDAP/IdM manager procedures (do NOT change local passwd when central auth is in place).

7.6 Local Windows machine account (cached credentials / local admin)

If the user uses a local Windows account, use lusrmgr.MSc on the machine (or via remote management) to reset the local password, or use Intune device actions for non-domain devices.

7.7 SaaS accounts

Follow vendor-specific processes. Common approach: Verify identity. Admin console -> Users -> Reset password / Send reset email. Require password change at next login and verify MFA settings.

8. Verification scripts & command examples

Keep a secure copy of approved automation scripts in the internal repo (access-controlled). Scripts must log actions and the operator who ran them.

Simple AD PowerShell snippet (logging example):

# WARNING: store logs securely
$operator = $env:USERNAME
$user = 'j.smith'
$pwd = ConvertTo-SecureString 'TempP@ssw0rd123!' -AsPlainText -Force
Set-ADAccountPassword -Identity $user -NewPassword $pwd -Reset
Set-ADUser -Identity $user -ChangePasswordAtLogon $true
"$(Get-Date) - $operator reset password for $user" | Out-File "C:\SecureLogs\pwdreset.log" -Append

9. Troubleshooting common scenarios

User still cannot sign in after reset:

  • Confirm they used the correct username and new password.
  • Verify the account isn't disabled.
  • If the device is offline, cached credentials may require the user to connect to the corporate network (VPN) to complete sign-in.
  • For MFA issues, guide on re-enrollment or escalate to IAM.

Password change loops / cannot change password at next logon:

  • Check User must change password at next logon flag in AD.
  • For Office 365, check sign-in logs for errors.

Account locked again immediately after reset:

  • Investigate the source of incorrect sign-ins (legacy devices, mapped drives, scheduled tasks, service accounts using old credentials).
  • Use sign-in logs and examine devices with stored credentials.

10. Escalation matrix & SLA

  • Tier 1: Password resets for standard users — SLA 15 minutes for verification + reset.
  • Tier 2: Complex resets, privileged accounts, devices with encryption — SLA 60 minutes.
  • Security / IAM: Incident investigation — SLA depends on severity; emergency response is immediate on-call.

Escalate if:

  • Reset cannot be completed due to a lack of rights.
  • There is suspicion of account compromise.
  • The account is privileged (Admin, Service Account, Executive) — follow privileged account procedure.

11. Audit, Logging & Compliance

All resets must be logged in the ticketing system with verification factors recorded. Administrative password changes performed via PowerShell or automation must have accompanying immutable logs. Periodic audits (quarterly) to validate resets and delegation.

12. Communication templates (email/ticket/chat)

Ticket note template (must be appended to ticket after reset):

Action: Password reset
Performed by: {agent_name}
Date/Time: {timestamp}
User verified by: {method1}, {method2}
Temporary password: NOT STORED here (masked)
User instructed to change password at next logon: Yes/No
Additional notes: {notes}

End-user instruction (after reset) — short version:

Hi {UserFirstName}, Your password was reset by the IT Service Desk at {timestamp} after verifying your identity. Please sign in using the temporary password we provided, and you will be prompted to set a new password. Make sure your new password satisfies company policy (minimum 12 characters). If you use mobile apps, email clients, or mapped drives, you may be asked to re-enter your credentials. If you did not request this reset, contact the Service Desk immediately.

13. Training & Quality Assurance

New agents must shadow password resets for 2 weeks and complete the internal Security & Verification course. Monthly QA review of a random sample of password reset tickets for verification of adequacy and documentation.

14. Metrics & Reporting

  • Number of password resets
  • Average time-to-reset
  • Number of resets requiring escalation
  • Number of post-reset incidents (suspected compromise)

1. Purpose & Scope

This manual provides standardized procedures for configuring corporate email accounts on mobile devices. It covers native mail clients (Apple Mail, Gmail, Samsung Mail), the recommended Microsoft Outlook app, and manual IMAP/SMTP configurations. The goal is to ensure secure, consistent, and compliant access to corporate email.

2. Roles & Responsibilities

  • Service Desk Agent (Tier 1): Guide users through email setup, provide troubleshooting, and verify connectivity.
  • Tier 2 Support / Messaging Admins: Handle escalations, advanced troubleshooting, MDM-related setups, and conditional access policies.
  • Security / IAM Team: Manage conditional access, MFA requirements, and mobile device compliance.
  • Managers: Approve exceptions (e.g., non-standard email apps).

3. Security & Compliance Policy

  • Only supported and approved apps should be used (Outlook mobile recommended).
  • MFA must be enabled for all mobile email accounts.
  • Email access requires device compliance (enrolled in Intune/MDM if mandated).
  • Prohibited: Third-party apps without approval, insecure IMAP/POP apps, or apps without encryption support.
  • Implement remote wipe access if device is lost/stolen (remote wipe via MDM or Outlook mobile app).

4. Supported Mobile Platforms & Apps

  • iOS (iPhone, iPad): Outlook, Apple Mail
  • Android: Outlook, Gmail, Samsung Mail
  • Other (limited support): Native IMAP/SMTP with prior approval

5. Prerequisites

  • User must know corporate email address and current password.
  • User's account must be MFA-enabled and registered for self-service password reset.
  • Device must meet compliance requirements (passcode enabled, encryption on, no rooted/jailbroken status).
  • If MDM-enrolled, ensure device is registered and compliant.

6. Standard Email Setup Workflows

6.1 iOS (Apple Mail)

Go to Settings > Mail > Accounts > Add Account. Choose Microsoft Exchange. Enter email and description. Authenticate with company login + MFA. Select services to sync (Mail, Contacts, Calendar).

6.2 Android (Gmail app)

Open Gmail > Add account > Exchange and Office 365. Enter email and authenticate with MFA. Accept security policies if prompted. Sync begins automatically.

6.3 Outlook for iOS/Android (recommended)

Download from App Store/Google Play. Launch app > Add Account. Enter email > authenticate with MFA. Corporate branding (if enabled) confirms successful login.

6.4 Manual IMAP/SMTP setup

Only if explicitly approved. IMAP: outlook.office365.com, Port 993, SSL/TLS. SMTP: smtp.office365.com, Port 587, STARTTLS.

6.5 MDM-enrolled device setup

If using Intune: Email profile may auto-configure. Confirm policy sync. If manual, follow standard setup with conditional access enforcement.

7. Step-by-Step Procedures

7.1 Outlook Mobile (preferred)

  1. Download Microsoft Outlook from App Store/Google Play.
  2. Open app > tap Add Account.
  3. Enter corporate email (e.g., user@company.com).
  4. Enter password > complete MFA challenge.
  5. If prompted, accept company security policies.
  6. Verify mail sync (send test email).

7.2 Apple Mail (iOS)

  1. Open Settings > Mail > Accounts > Add Account.
  2. Select Microsoft Exchange.
  3. Enter email and account description.
  4. Tap Sign In > enter password and complete MFA.
  5. Select services (Mail, Contacts, Calendars, Reminders).
  6. Open Mail app and confirm sync.

7.3 Gmail App (Android)

  1. Open Gmail > Settings > Add account.
  2. Choose Exchange and Office 365.
  3. Enter corporate email.
  4. Authenticate with password + MFA.
  5. Accept device security requirements.
  6. Confirm sync by sending/receiving.

7.4 Samsung Mail App

  1. Open Settings > Accounts and backup > Accounts > Add account.
  2. Select Email.
  3. Choose Office 365/Exchange.
  4. Enter corporate email and password.
  5. Authenticate with MFA.
  6. Confirm sync.

7.5 Manual IMAP/SMTP configuration

Note: Use only if Exchange ActiveSync/Outlook mobile is not available.

  • Incoming: outlook.office365.com (Port 993, SSL/TLS)
  • Outgoing: smtp.office365.com (Port 587, STARTTLS)
  • Username: full email address
  • MFA users must use app passwords (if legacy auth allowed — otherwise blocked)

8. MFA & Conditional Access Considerations

  • Users may be required to use Outlook mobile exclusively if conditional access enforces modern authentication.
  • Jailbroken/rooted devices will be blocked.
  • Ensure users have Authenticator app installed for MFA prompts.

9. Troubleshooting Common Issues

Problem: Email not syncing.

  • Check Wi-Fi/data connection.
  • Verify password/MFA success.
  • Remove/re-add account.
  • For MDM devices, force sync policy.

Problem: MFA prompt not appearing.

  • Ensure Authenticator app is installed.
  • Check if SMS/alternate method is enabled.

Problem: Mail app says "account not verified."

  • Ensure correct server type (Exchange, not IMAP).
  • Check conditional access logs in Azure AD.

10. Escalation Matrix & SLA

  • Tier 1: Setup guidance and basic troubleshooting — SLA 15 minutes.
  • Tier 2: Escalation for MDM, policy blocks, advanced issues — SLA 1 hour.
  • IAM/Security: Escalation for conditional access blocks, compromised accounts — immediate response.

11. Audit, Logging & Compliance

All mobile email setups should be logged in the ticket system. Record device type, OS version, client used, and date/time. Ensure compliance logs (Intune/Azure AD sign-in logs) reflect access.

12. Communication Templates

Ticket note template:

Action: Mobile email setup
Performed by: {agent_name}
Device: {iOS/Android + version}
Client: {Outlook/Apple Mail/Gmail}
Date/Time: {timestamp}
Verification: User identity confirmed
Outcome: Setup successful / pending sync
Notes: {additional info}

End-user instruction:

Hi {UserFirstName}, Your corporate email has been successfully set up on your {device_type}. Please ensure: Your device lock screen and passcode remain enabled. You update your password on the device when you next change your corporate password. Notify IT immediately if your device is lost or stolen. If you experience issues, reply to this ticket or contact the Service Desk.

13. Training & Quality Assurance

All Service Desk agents must practice setup on demo devices. Monthly QA reviews will sample tickets for proper documentation.

14. Metrics & Reporting

  • Number of mobile email setups per week.
  • Percentage of users on Outlook vs. native clients.
  • Average resolution time.
  • Failed setup/escalation rate.

1. Introduction

This guide provides step-by-step instructions to help users connect to Wi-Fi networks, diagnose issues, and apply troubleshooting methods across devices (Windows, macOS, iOS, Android). It also includes advanced checks for IT support staff and escalation procedures.

2. General Prerequisites

Before attempting to connect:

  • Ensure the Wi-Fi router/access point is powered on.
  • Confirm the user has the Wi-Fi network name (SSID) and password.
  • Check that the device's Wi-Fi adapter is turned on.
  • Verify that airplane mode is turned off.

3. Connecting to Wi-Fi

3.1 Windows 10/11

  1. Click the Wi-Fi icon in the system tray.
  2. Select the correct SSID from the list.
  3. Click Connect.
  4. Enter the Wi-Fi password.
  5. Confirm you are connected (status: "Connected, secured").

3.2 macOS

  1. Click the Wi-Fi icon in the menu bar.
  2. Select the desired network.
  3. Enter the Wi-Fi password if prompted.
  4. Check for a connected status and internet availability.

3.3 iOS (iPhone/iPad)

  1. Open Settings → Wi-Fi.
  2. Toggle Wi-Fi On.
  3. Tap the desired network.
  4. Enter password → tap Join.
  5. Confirm connection (checkmark next to SSID).

3.4 Android

  1. Open Settings → Network & Internet → Wi-Fi.
  2. Toggle Wi-Fi On.
  3. Select the correct network.
  4. Enter password → tap Connect.
  5. Confirm status: "Connected."

4. Basic Troubleshooting Steps

4.1 Quick Fixes

  • Restart the device.
  • Toggle Wi-Fi off and back on.
  • Forget and re-add the Wi-Fi network.
  • Restart the router/modem (if at home or authorized).
  • Check other devices to confirm if it's a device issue or network issue.

4.2 Signal Issues

  • Move closer to the Wi-Fi router.
  • Minimize interference (microwaves, cordless phones, thick walls).
  • Switch to the 5GHz band if available for faster, more stable connections.

4.3 Password Issues

  • Double-check case-sensitive characters.
  • Confirm with IT/admin if the password has been updated.
  • For corporate networks, ensure correct domain credentials are entered.

5. Intermediate Troubleshooting

5.1 Windows

  • Run Windows Network Troubleshooter:
    • Right-click Wi-Fi icon → Troubleshoot problems.
  • Reset TCP/IP and DNS cache:
    • Open Command Prompt (Admin).
    • Run: ipconfig /release, ipconfig /renew, ipconfig /flushdns
  • Disable and re-enable the wireless adapter.

5.2 macOS

  • Go to System Preferences → Network.
  • Remove and re-add the Wi-Fi service.
  • Use the Wireless Diagnostics tool: Hold Option → click Wi-Fi icon → select Open Wireless Diagnostics.

5.3 Mobile Devices (iOS/Android)

  • Toggle Airplane Mode on and off.
  • Reset network settings:
    • iOS: Settings → General → Reset → Reset Network Settings.
    • Android: Settings → System → Reset Options → Reset Wi-Fi, mobile & Bluetooth.
  • Update device software.

6. Advanced Troubleshooting (IT Support Staff)

6.1 Network Configuration Checks

Check IP configuration:

  • Windows: Run ipconfig /all to verify IP, DNS, and gateway.
  • macOS/Linux: Run ifconfig or ip a.

6.2 Connectivity Tests

Ping tests:

  • ping 8.8.8.8 → test internet connectivity.
  • ping gateway_IP → test router connection.

6.3 DNS Resolution

DNS issues: Change DNS to Google (8.8.8.8, 8.8.4.4) or Cloudflare (1.1.1.1).

6.4 Driver and Firmware Updates

Driver/Firmware updates:

  • Update wireless network drivers (Windows Device Manager).
  • Update router firmware.

6.5 Network Access Controls

  • Check for MAC address filtering on router.
  • Hidden SSID: Manually add SSID and password.

7. Common Error Messages & Fixes

  • "Incorrect password": Likely Cause: Wrong password or expired credentials. Solution: Verify with IT/admin.
  • "Connected, no internet": Likely Cause: Router issue, DHCP error, DNS problem. Solution: Restart router, reset IP, change DNS.
  • "Authentication error": Likely Cause: Wrong security type or credentials. Solution: Re-enter credentials, check WPA2/WPA3 compatibility.
  • "Unable to join network": Likely Cause: Signal issue, router limit reached. Solution: Move closer, restart router, check device count.
  • "IP address not assigned": Likely Cause: DHCP issue. Solution: Set static IP or renew DHCP lease.

8. Best Practices & Security

8.1 Security Standards

  • Use WPA2/WPA3 encryption (never WEP).
  • Disable auto-connect to open networks.
  • Regularly update router firmware and device OS.
  • Change Wi-Fi passwords periodically.

8.2 Corporate Network Requirements

For corporate networks:

  • Use enterprise authentication (802.1X).
  • Enforce MFA for VPN/Wi-Fi logins.
  • Implement network segmentation (guest vs. internal Wi-Fi).

9. Escalation Guidelines

9.1 When to Escalate

Escalate the issue to Tier 2/Network team if:

  • Multiple users are affected on the same network.
  • Device drivers and OS are up to date but issues persist.
  • DNS or DHCP server errors are recurring.
  • Hardware failure is suspected (router, access point, or NIC).

9.2 Information to Collect Before Escalation

  • Device type, OS version, and Wi-Fi adapter details.
  • SSID, error messages, and troubleshooting steps already performed.
  • Ping/traceroute results.
  • Date/time the issue started.

10. Quick Reference Commands

10.1 Windows Commands

netsh wlan show profile         # Show saved Wi-Fi profiles
netsh winsock reset            # Reset Winsock

ip a                          # View network interfaces (Linux)
ping google.com               # Test connectivity
sudo dscacheutil -flushcache  # Flush DNS (macOS)

1. Introduction

This manual provides guidance for setting up printers (local, network, and wireless) and troubleshooting common printing problems across Windows, macOS, and mobile devices. It is intended for end-users and IT support staff.

2. Printer Setup

2.1 Local Printer (USB) Setup

  • Connect the printer to the computer via USB cable.
  • Power on the printer.
  • Windows:
    • Go to Settings → Devices → Printers & Scanners → Add a printer or scanner.
    • Select the printer and click Add Device.
    • Install drivers if prompted.
  • macOS:
    • Go to System Settings → Printers & Scanners → Add Printer.
    • Select the USB printer from the list.
    • Install required drivers.

2.2 Network (Ethernet) Printer Setup

  • Connect printer to the LAN via Ethernet cable.
  • Find printer's IP address (via printer display or configuration sheet).
  • Windows:
    • Go to Printers & Scanners → Add a printer or scanner → The printer I want isn't listed.
    • Select Add a printer using TCP/IP address.
    • Enter the IP address and install drivers.
  • macOS:
    • Go to Printers & Scanners → Add Printer.
    • Select IP tab → enter IP address.
    • Choose protocol: HP Jetdirect – Socket (most common).
    • Install drivers.

2.3 Wireless (Wi-Fi) Printer Setup

  • On the printer, enable Wireless Setup Wizard.
  • Connect to the correct Wi-Fi SSID (enter password).
  • Print a network configuration page to verify connection.
  • Windows/macOS: Add printer as a network printer (it should appear automatically if on the same Wi-Fi).
  • For home printers, you can also use the manufacturer's setup utility/app (e.g., HP Smart, Canon IJ, Epson Connect).

2.4 Mobile Devices (iOS/Android)

  • iOS: Supports AirPrint. Open the app → Share/Print → select AirPrint-enabled printer.
  • Android: Use Default Print Service or manufacturer app (HP Print Service, Canon Print, Epson iPrint).
  • Ensure both device and printer are on the same Wi-Fi.

3. Common Printer Problems and Fixes

3.1 Printer Not Detected

  • Ensure printer is powered on and connected.
  • Check USB cable or Wi-Fi/Ethernet connection.
  • Restart printer and computer.
  • For Wi-Fi printers: confirm both are on the same network.

3.2 Printer Offline

  • Windows: Go to Control Panel → Devices & Printers → Right-click printer → uncheck Use Printer Offline.
  • macOS: Go to Printers & Scanners → select printer → resume queue.
  • Restart spooler service (Windows): Run → services.msc → restart Print Spooler.

3.3 Paper Jams

  • Turn off printer, open access panels, and gently remove jammed paper.
  • Check for torn bits inside.
  • Reload paper correctly.

3.4 Poor Print Quality

  • Run clean printhead or nozzle check from printer settings.
  • Replace low ink/toner cartridges.
  • Use correct paper type and quality settings.

3.5 Slow Printing

  • Reduce print resolution (draft mode).
  • Ensure printer has enough memory (for large documents).
  • Use wired connection instead of Wi-Fi if possible.

3.6 Driver Issues ("Driver Unavailable" or Wrong Driver)

  • Download and install latest drivers from manufacturer's website.
  • Remove old drivers: Windows: Device Manager → Printers → Uninstall device.
  • macOS: Remove printer and re-add with updated driver.

4. Advanced Troubleshooting

4.1 Network Printer IP Issues

  • Ping the printer's IP: ping [IP address].
  • If unreachable, confirm printer is on correct subnet.
  • Assign a static IP to avoid future conflicts.

4.2 Print Spooler Issues (Windows)

  • Run Command Prompt as Admin.
  • Enter: 
    net stop spooler
del %systemroot%\System32\spool\PRINTERS\* /Q
net start spooler
  • Re-add printer.

4.3 Firewall/Antivirus Blocking

  • Temporarily disable firewall/antivirus to test.
  • Add printer software to exceptions list if blocking detected.

4.4 Shared Printers (Corporate Environments)

  • Ensure correct permissions are granted.
  • Access printer via UNC path: \\servername\printername.
  • Confirm group policy deployment for printers is applied.

6. Preventive Maintenance & Best Practices

  • Regularly update printer firmware and drivers.
  • Use manufacturer-approved cartridges/toners.
  • Keep printers clean; remove dust and paper fragments.
  • Avoid overloading trays with paper.
  • Educate users on correct paper orientation and handling.
  • Assign static IPs for network printers to prevent conflicts.

7. Escalation Guidelines

7.1 When to Escalate

  • Printer hardware malfunction (e.g., broken fuser, rollers).
  • Recurring spooler crashes.
  • Network-wide printing issues.
  • Enterprise print server errors.
  • Specialized printers (label/plotter) requiring vendor servicing.

7.2 Information to Collect Before Escalation

  • Printer model and serial number.
  • Connection type (USB, Wi-Fi, Ethernet).
  • OS version of affected device(s).
  • Error codes/messages.
  • Steps already attempted.

1. Introduction

A Virtual Private Network (VPN) enables secure, encrypted connections between a user's device and the corporate network or internet. This guide provides step-by-step instructions for connecting to VPNs across common platforms, troubleshooting common issues, and escalation paths for IT support.

2. General Prerequisites

Before connecting to a VPN, ensure the following:

  • Active internet connection (Wi-Fi, Ethernet, or mobile data).
  • VPN client software installed (e.g., Cisco AnyConnect, FortiClient, OpenVPN, GlobalProtect).
  • Correct VPN server address (URL or IP).
  • Valid user credentials (username, password, and possibly MFA/OTP).
  • Any required security certificates installed.
  • Administrative permissions if required for client installation.

3. VPN Setup and Connection

3.1 Windows 10/11 (Built-in VPN Client)

  • Go to Settings → Network & Internet → VPN → Add a VPN connection.
  • Select provider: Windows (built-in).
  • Enter details:
    • VPN provider: Windows (built-in)
    • Connection name: [Company VPN]
    • Server name or address: [vpn.company.com]
    • VPN type: Automatic / L2TP / IKEv2 / PPTP (as specified by IT)
    • Type of sign-in: Username and password or certificate
  • Click Save.
  • Connect via Wi-Fi/Ethernet icon → VPN.

3.2 macOS

  • Go to System Settings → Network.
  • Click + → select VPN interface.
  • Enter configuration details:
    • VPN type: (IKEv2, L2TP/IPSec, Cisco IPSec).
    • Server address.
    • Username and password.
  • Apply settings.
  • Click Connect from the Network menu bar.

3.3 iOS (iPhone/iPad)

  • Go to Settings → General → VPN & Device Management → Add VPN Configuration.
  • Choose protocol (IKEv2, IPSec, L2TP).
  • Enter:
    • Server address
    • Remote ID (if required)
    • Username & password
  • Save.
  • Toggle VPN On from main settings.

3.4 Android

  • Go to Settings → Network & Internet → VPN → Add VPN.
  • Enter VPN name, type, and server address.
  • Input username & password.
  • Save.
  • Select the VPN to connect.

3.5 Third-Party VPN Clients (Corporate Use)

  • Cisco AnyConnect: Install → enter VPN server → log in with credentials/MFA.
  • FortiClient: Install → configure server IP and credentials → connect.
  • OpenVPN: Import .ovpn configuration file from IT → log in → connect.
  • Palo Alto GlobalProtect: Enter portal address → log in → connect.

4. Common VPN Problems and Fixes

4.1 VPN Won't Connect

  • Confirm internet connection works outside VPN.
  • Double-check server address and login credentials.
  • Restart device and VPN client.
  • If on Wi-Fi, try switching to Ethernet.

4.2 Slow VPN Connection

  • Connect to the nearest VPN server.
  • Close bandwidth-heavy apps (streaming, large downloads).
  • Switch from Wi-Fi to wired connection.
  • Change VPN protocol (e.g., switch from UDP to TCP in OpenVPN).

4.3 Authentication Errors

  • Verify username and password are correct.
  • Check if MFA/OTP token is required.
  • Reset expired credentials or request a password reset.

4.4 DNS/Internet Issues After Connecting

If connected but no internet:

  • Flush DNS: ipconfig /flushdns (Windows).
  • Change DNS to Google (8.8.8.8, 8.8.4.4) or Cloudflare (1.1.1.1).
  • Split tunneling issues: confirm corporate IT policy.

4.5 Blocked by Firewall/ISP

  • Test connection on a different network.
  • Ask IT if specific ports/protocols are required (e.g., UDP 500/4500 for IPSec).
  • Ensure firewall or antivirus isn't blocking VPN client.

4.6 Certificate Errors

  • Reinstall/update required certificates.
  • Confirm system clock and time zone are correct.

5. Advanced Troubleshooting (IT Staff)

5.1 Network Configuration Checks

Check IP assignment:

  • Windows: ipconfig /all
  • Linux/macOS: ifconfig or ip a

5.2 Connectivity Tests

  • Ping gateway: Verify VPN tunnel endpoint connectivity.
  • Traceroute: Identify where traffic is dropping.

5.3 Log Analysis

Review VPN client logs:

  • Cisco AnyConnect: C:\ProgramData\Cisco\Cisco AnyConnect Secure Mobility Client\Logs
  • OpenVPN: Client log in GUI or openvpn.log file.

5.4 Client and Authentication Issues

  • Reinstall VPN client if corruption suspected.
  • Verify AD/LDAP authentication if corporate domain login is required.

6. Best Practices & Security

6.1 Security Standards

  • Always use the official company VPN client or configuration.
  • Never share VPN credentials.
  • Keep the VPN client and OS up to date.
  • Use MFA wherever possible.
  • Disconnect when not in use to save bandwidth and battery.

6.2 Corporate Network Requirements

For corporate networks:

  • Enforce split tunneling policy.
  • Use endpoint security tools to prevent unauthorized access.

8. Escalation Guidelines

8.1 When to Escalate

Escalate to Tier 2 or Network team if:

  • VPN service is down for multiple users.
  • Connection fails despite correct credentials and client.
  • Persistent performance issues not resolved by client-side troubleshooting.
  • Certificates or authentication servers appear to be failing.

8.2 Information to Collect Before Escalation

  • Server name/IP.
  • Error messages/screenshots.
  • Logs (if accessible).
  • Date/time issue started.

1. Introduction

Modern browsers like Google Chrome, Microsoft Edge, and Apple Safari are essential for accessing web applications, intranet systems, and cloud services. Users may encounter issues such as slow performance, page loading errors, crashes, or login failures. This manual provides step-by-step troubleshooting, configuration checks, and escalation guidelines.

2. Common Issues and Symptoms

  • Pages not loading (error messages like ERR_CONNECTION_TIMED_OUT).
  • Slow browsing performance.
  • Frequent crashes or freezing.
  • Issues with cached data or cookies.
  • Extension conflicts.
  • Login/authentication problems with corporate apps.
  • SSL/TLS certificate warnings.
  • Browser not syncing bookmarks/passwords.
  • Compatibility issues with web apps (e.g., some features not working in Safari).

3. Initial Checks (All Browsers)

3.1 Confirm Internet Connectivity

  • Test Wi-Fi or LAN connection.
  • Open another website (e.g., www.google.com).

3.2 Check Corporate VPN / Proxy Settings

  • Disconnect/reconnect VPN.
  • Ensure proxy settings are correct.

3.3 Basic Browser Restart

  • Close all tabs, relaunch the browser.

3.4 System Reboot

  • Restart the computer or mobile device.

4. Google Chrome Troubleshooting

4.1 Basic Troubleshooting

  • Update Chrome: Menu → Help → About Google Chrome.
  • Clear browsing data: Ctrl + Shift + Del → Clear cache, cookies.
  • Disable extensions: Menu → More tools → Extensions.
  • Reset settings: Menu → Settings → Reset settings → Restore settings to default.

4.2 Advanced Fixes

  • Flush DNS cache: chrome://net-internals/#dns → Clear host cache.
  • Hardware acceleration: Disable under Settings → System.
  • Create new Chrome profile: Menu → Settings → You and Google → Add new profile.

5. Microsoft Edge Troubleshooting

5.1 Basic Troubleshooting

  • Update Edge: Menu → Help and feedback → About Microsoft Edge.
  • Clear cache/cookies: Ctrl + Shift + Del.
  • Disable extensions: Menu → Extensions → Manage extensions.
  • Reset settings: Settings → Reset settings → Restore settings to default.

5.2 Advanced Fixes

  • Reset browser flags: edge://flags → Reset all.
  • Check site permissions: Lock icon → Site permissions.
  • Use InPrivate mode for login issues: Ctrl + Shift + N.

6. Apple Safari Troubleshooting (macOS & iOS)

6.1 Basic Troubleshooting

  • Clear history/cache: Safari → History → Clear History.
  • Disable extensions: Safari → Preferences → Extensions.
  • Ensure latest macOS/iOS update installed.
  • Check "Develop" menu: Enable via Preferences → Advanced → Show Develop menu.

6.2 Advanced Fixes

  • Empty caches: Develop → Empty Caches.
  • Disable experimental features: Develop → Experimental Features → Reset.
  • Reset Safari: Remove preference files:
    macOS: ~/Library/Preferences/com.apple.Safari.plist

7. Common Authentication Issues

  • Clear cookies for the specific site (common with Office 365 login loops).
  • Try different browser (cross-test Chrome/Edge/Safari).
  • Ensure system clock is accurate (SSL/TLS errors often caused by wrong time).
  • Check if Multi-Factor Authentication (MFA) is blocking login.

8. SSL/TLS Certificate Errors

  • NET::ERR_CERT_DATE_INVALID → Check system date/time.
  • NET::ERR_CERT_AUTHORITY_INVALID → Corporate SSL certificate may need to be installed.
  • If on VPN, verify security certificates are updated.

9. Browser Performance Optimization

  • Keep browsers updated.
  • Limit open tabs and extensions.
  • Clear cache periodically.
  • Use Task Manager (Chrome: Shift + Esc, Edge: Shift + Esc) to identify high-resource tabs.
  • Disable unused plugins.

10. Escalation Guidelines

10.1 When to Escalate

  • Browser crashes repeatedly after reset.
  • SSL errors persist across multiple browsers.
  • Corporate apps fail due to browser compatibility.
  • Suspected malware or persistent unwanted extensions are found.

11. User Education

  • Avoid installing unknown extensions.
  • Do not disable browser security warnings.
  • Use official download links for browser updates.
  • Regularly sign out from shared/public computers.

1. Introduction

A slow computer can impact productivity, cause user frustration, and affect system stability. Performance issues may be caused by insufficient resources, unnecessary applications, malware, outdated software, or hardware bottlenecks. This manual provides step-by-step troubleshooting and fixes for both Windows and macOS systems, designed for end-users and IT support teams.

2. Common Symptoms

  • Programs take long to open or respond.
  • System startup/shutdown is slow.
  • Frequent freezing or "Not Responding" messages.
  • High CPU, memory, or disk usage.
  • Fan constantly running or device overheating.
  • Lag during video calls, streaming, or multitasking.

3. Initial Checks

3.1 Basic System Restart

Restart the Computer – clears temporary memory and resets processes.

3.2 Network Connectivity

Check Internet Speed – slowness may be due to poor connectivity.

3.3 Storage Verification

Confirm Storage Availability – ensure at least 15–20% free disk space.

3.4 Resource Monitoring

Check Task/Activity Monitor – identify resource-heavy processes:

  • Windows: Ctrl + Shift + Esc → Task Manager.
  • macOS: Applications → Utilities → Activity Monitor.

4. Windows Troubleshooting Steps

4.1 Basic Fixes

  • Run **Windows Update (Settings → Update & Security).
  • Restart and install pending updates.
  • Close unnecessary background apps.
  • Uninstall unused software (Control Panel → Programs → Uninstall a program).

4.2 Startup Optimization

1. Overview

File sharing enables users to collaborate by granting controlled access to files and folders. Proper permission management ensures secure collaboration while maintaining data integrity and compliance with corporate security policies.

This manual provides comprehensive guidance for configuring and troubleshooting file sharing across Windows, macOS, and cloud platforms.

Important: All file sharing must comply with corporate data classification and security policies.

2. Key Concepts and Terminology

Permission Levels

  • Read: View and open files, copy files
  • Write/Modify: Create, edit, and delete files and folders
  • Full Control: All modify permissions plus change permissions and take ownership
  • List Folder Contents: View folder contents without opening files

Key Terms

  • Share Permissions: Control network access to shared folders
  • NTFS Permissions: Control local access to files and folders (Windows)
  • Inheritance: Child objects automatically receive parent permissions
  • Effective Permissions: Combination of all applicable permissions
  • Principal: User, group, or computer account that can be assigned permissions

Permission Interaction (Windows)

  • Most Restrictive Rule: When both Share and NTFS permissions apply, the most restrictive takes precedence
  • Local vs Network: NTFS permissions apply to both local and network access; Share permissions only apply to network access

3. Windows File Sharing

3.1 Basic Folder Sharing

Prerequisites

  • Administrator rights or ownership of the folder
  • Network discovery enabled
  • File and printer sharing enabled in Windows Firewall

Steps

  1. Right-click the target folder
  2. Select Properties
  3. Navigate to Sharing tab
  4. Click Share...
  5. In the File Sharing dialog:
    • Enter username, group name, or email address
    • Click Add
    • Select permission level from dropdown:
      • Read: View-only access
      • Read/Write: Full modify access
  6. Click Share
  7. Note the network path (e.g., `\\ComputerName\ShareName`)
  8. Click Done

Verification

  • Test access from another computer using the network path
  • Verify permissions work as expected

3.2 Advanced Sharing Configuration

When to Use Advanced Sharing

  • Need to limit concurrent users
  • Require custom share names
  • Need granular permission control
  • Corporate compliance requirements

Steps

  • Right-click folder → Properties → Sharing tab
  • Click Advanced Sharing...
  • Check Share this folder
  • Configure options:
    • Share name: Custom network name for the share
    • Comments: Description for the share
    • User limit: Maximum concurrent connections
  • Click Permissions to configure access:
    • Add/Remove: Manage users and groups
    • Permission levels: Full Control, Change, Read
  • Apply settings and close dialogs

3.3 NTFS Security Permissions

Purpose

NTFS permissions provide granular security control for local and network access.

Configuration Steps

  • Right-click folder → Properties → Security tab
  • Review existing permissions
  • To modify permissions:
    • Select user/group → Click Edit
    • Or click Advanced for detailed control
  • Set permissions:
    • Full Control: Complete access including permission changes
    • Modify: Read, write, delete files and subfolders
    • Read & Execute: View and run files
    • List Folder Contents: View folder contents
    • Read: View files and properties
    • Write: Create files and write data
  • Apply changes

Permission Inheritance

  • Enabled by default: Child objects inherit parent permissions
  • To modify: **Advanced Security Settings** → **Disable inheritance**
  • Best practice: Use inheritance for consistent security

Remote Desktop Connection (RDC) enables users to access and control computers over a network connection as if they were physically present at the remote machine. This technology is essential for remote work, IT support, and accessing corporate resources from external locations.

This manual provides comprehensive guidance for configuring, connecting to, and troubleshooting Remote Desktop connections across multiple platforms.

Important: All Remote Desktop connections must comply with corporate security policies and be established through approved VPN connections when accessing from external networks.

2. Key Concepts and Terminology

Core Components

  • Remote Desktop Protocol (RDP): Microsoft's proprietary protocol for remote access (default port 3389)
  • Host Computer: The target machine being accessed remotely
  • Client Device: The device initiating the remote connection
  • Session: An active remote desktop connection instance

Authentication Methods

  • Local Account: Username/password from the host computer's local user database
  • Domain Account: Active Directory credentials for domain-joined computers
  • Network Level Authentication (NLA): Enhanced security requiring authentication before session establishment

Connection Types

  • LAN Connection: Direct network access within the same subnet
  • VPN Connection: Secure tunnel through corporate VPN infrastructure
  • Internet Connection: Direct internet access (not recommended for corporate use)

3. System Requirements

Host Computer Requirements

  • Operating System: Windows 10 Pro/Enterprise, Windows 11 Pro/Enterprise, Windows Server editions
  • Network: Active network connection with reachable IP address
  • Power State: Computer must be powered on (Wake-on-LAN may be configured)
  • User Account: Valid local or domain user account with appropriate permissions
  • Firewall: Windows Firewall configured to allow Remote Desktop connections

Client Device Requirements

  • Windows: Built-in Remote Desktop Connection client or Windows Store app
  • macOS: Microsoft Remote Desktop from Mac App Store
  • iOS/Android: Microsoft Remote Desktop mobile app
  • Network: Network connectivity to host (LAN, VPN, or internet)

Network Requirements

  • Port Access: TCP port 3389 accessible between client and host
  • Bandwidth: Minimum 1 Mbps for basic functionality, 5+ Mbps for optimal experience
  • Latency: Less than 150ms for responsive interaction
  • VPN: Required for external connections per corporate security policy

8. Security Best Practices

Host Configuration

  • Regular updates: Keep Windows and RDP components updated
  • Strong authentication: Use complex passwords and consider smart cards
  • User access control: Limit Remote Desktop users to necessary personnel only
  • Session timeouts: Configure automatic disconnection for idle sessions
  • Audit logging: Enable and monitor Remote Desktop connection logs

Network Security

  • VPN requirement: Never expose RDP directly to the internet
  • Port security: Change default port 3389 if required by security policy
  • Network segmentation: Isolate Remote Desktop traffic where possible
  • Monitoring: Implement network monitoring for RDP connections

Client Security

  • Trusted connections: Only connect to authorized and trusted hosts
  • Certificate validation: Verify server certificates to prevent man-in-the-middle attacks
  • Resource sharing: Be cautious when sharing local drives and clipboard
  • Session management: Always disconnect properly, never just close the client window

Corporate Compliance

  • Data classification: Ensure remote access complies with data handling policies
  • Regulatory requirements: Meet industry-specific compliance standards
  • Privacy protection: Implement controls for personal data access
  • Incident response: Report security incidents involving Remote Desktop immediately

This manual provides standardized procedures for installing approved software across corporate devices. Following these guidelines ensures:

  • Compliance with organizational security and licensing policies
  • Consistent software configurations across the enterprise
  • Reduced installation-related support incidents
  • Proper documentation and audit trails

System Prerequisites

  • Administrative privileges on the target device
  • Sufficient system resources:
    • Available disk space (minimum 2x software size)
    • RAM meeting minimum requirements
    • Compatible operating system version
  • Network connectivity for download and activation
  • Current system backups (recommended for critical systems)

Approval and Documentation

  • Software approval from IT Security and Compliance team
  • Valid license or enterprise agreement coverage
  • Change request ticket (for production systems)
  • Installation documentation requirements met

Pre-Installation Checklist

  • Verify software is on approved applications list
  • Confirm system meets minimum requirements
  • Check for existing versions or conflicting software
  • Ensure user has appropriate permissions
  • Schedule installation during approved maintenance windows (if required)
  • Prepare rollback plan if installation fails

4. Windows Software Installation

4.1 Manual Installation from Installer Files

Prerequisites

  • Administrator rights on the local machine
  • Installer file from approved source
  • Antivirus software updated and active

Installation Process

  1. Download installer from official vendor website or company portal.
  2. Verify file integrity:
    • Check digital signature: Right-click file → Properties → Digital Signatures
    • Scan with antivirus if required
  3. Run installer with elevated privileges:
    • Right-click installer file → Run as administrator
    • Or use PowerShell: Start-Process -FilePath "installer.exe" -Verb RunAs
  4. Follow installation wizard:
    • Review and accept End User License Agreement (EULA)
    • Choose installation type:
      • Typical: Standard features and default location
      • Custom: Select specific features and installation path
    • Select destination folder (use default unless specific requirements)
    • Configure additional options as required
  5. Complete installation:
    • Click Install to begin process
    • Wait for installation completion
    • Restart computer if prompted
  6. Verify installation:
    • Launch application to confirm functionality
    • Check Start Menu or desktop shortcuts
    • Verify licensing activation if required

Post-Installation Tasks

  • Update software to latest version if needed
  • Configure application settings per corporate standards
  • Test basic functionality
  • Document installation in asset management system

7. License Activation and Management

License Types

  • Volume Licensing: Enterprise agreements with activation servers
  • Individual Licenses: Product keys for single installations
  • Subscription Licenses: Cloud-based licensing (Office 365, Adobe CC)
  • Open Source: No licensing requirements but compliance is needed

Activation Methods

  • Online Activation
    1. Launch the application after installation.
    2. Enter the product key when prompted.
    3. The application connects to the vendor's activation server.
    4. The license is validated and applied automatically.
    5. Verify activation status in the application's settings.
  • Offline Activation
    1. Generate an activation request file.
    2. Transfer it to an internet-connected computer.
    3. Submit the request to the vendor's activation website.
    4. Download the activation response file.
    5. Apply the response file to the offline computer.
  • Enterprise Activation
    • KMS (Key Management Service): Automatic activation for domain computers
    • MAK (Multiple Activation Key): Phone or online activation for a specific number of installs
    • Subscription Services: Cloud-based activation through organizational accounts

License Compliance

  • Maintain accurate software inventory
  • Track license usage against purchased quantities
  • Regular compliance audits and reporting
  • Proper license documentation and storage
